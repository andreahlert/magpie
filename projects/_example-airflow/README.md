# Example adopter: Apache Airflow shape

Reference materialised intent + lock for an adopter whose workflow resembles Apache Airflow. **Illustrative**, not the real Airflow PMC's adopted config. URLs and tracker repo are concrete examples; substitute your project's values when copying.

Use this directory as the starting point when onboarding a project with:

- An active security flow (private list + CVE allocation).
- Large PR queue with triage + mentorship needs.
- Public mailing list archived in Ponymail.
- Comfortable with agent-drafted PRs under maintainer review (`risk-tier-max: draft-pr`).

## Files

| File | Purpose |
|---|---|
| `.apache-steward.intent.yaml` | Source of truth. Edit this. |
| `.apache-steward.lock` | Generated by `magpie apply`. Commit. |
| `overrides/` | (Optional) Per-skill template / param overrides beyond what fits in `intent.overrides.params`. Empty in this example. |

## Reproducing the lock

```bash
cd /path/to/your-adopter-repo  # or this directory for the example
uv run --with pyyaml --with jsonschema --with jinja2 \
  python -m reconciler apply \
    --intent .apache-steward.intent.yaml \
    --lock   .apache-steward.lock \
    --experimental \
    --stable-timestamp
```

`--stable-timestamp` fixes `generated-at` to `1970-01-01T00:00:00Z` so the committed example stays reproducible across re-renders. Real adopters omit `--stable-timestamp` and let the timestamp reflect the actual generation time.

## What this intent demonstrates

- **Five domains enabled** — security, pr-queue, issue-queue, contributor-lifecycle, framework-meta.
- **Risk ceiling at `draft-pr`** — auto-merge stays excluded by design.
- **Skill exclusions** — `issue-reassess` and `issue-reassess-stats` excluded because reassessment campaigns are on-demand for this project; the skills can be force-included on the rare sweep.
- **Pin** — `security-issue-import` pinned to 1.0.0 to demonstrate the pin override mechanism.
- **Params** — `pr-management-triage` and `security-issue-import` carry per-skill parameters that render into the skill's templates.

## What this is NOT

- The Airflow PMC has not formally adopted this configuration. The URLs are real Apache Airflow documentation URLs to make the example look concrete.
- A drop-in for another project. Copy and substitute.
